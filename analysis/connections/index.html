  1. Все события помечаем идентификатором, чтобы можно было видеть, что они относятся к одной связи. Category: client-connection
    Action: <название действия>-<идентификатор>
    Label: create-link
  2. Дублируем все события без идентификатора, чтобы без фильтров можно было понять динамику по каждому из них. Category: client-connection
    Action: <название действия>
    Label: create-link
  3. Все события отправляем по какому-то завершающему действию (сабмит, показать, т.д.), чтобы не засорять аналитику тем, что пользователь перетыкивает по 10 раз. Нас интересует конечный набор параметров. Исключения из этого правила буду описаны и помечены .
